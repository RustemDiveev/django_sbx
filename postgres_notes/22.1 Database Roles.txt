PostgreSQL управляет разрешениями доступа БД через роли. Роль можно представить как пользователя БД, или как группу пользователей БД в зависимости от настройки роли. Роли могут быть владельцами объектов БД (например, таблицы и функции) или могут назначать привилегии на эти объекты другим ролям, для контроля доступа к объектам. При этом, возможно задать членство в роли другой роли, позволяя дочерней роли использовать привилегии, назначенные родительской роли. 

Концепт роли включает в себя концепты "Пользователей" и "Групп". В версиях до 8.1 были пользователи и группы - разные типы сущностей, сейчас только роли. Каждая роль может вести себя как пользователь, как группа, или как то и другое. 

Роли БД концептуально отличаются от пользователей ОС. На практике может быть удобно соблюдать соответствие, но это необязательно. Роли БД глобальны для установленного кластера БД (не для отдельной БД). 

Для создания роли нужно выполнить CREATE ROLE: 
CREATE ROLE name;

name должно следовать правилам идентификаторов SQL: без спецсимволов или с двойным кавычками. На практике к CREATE ROLE захочется добавить дополнительные настройки (например, LOGIN). 

Для удаления роли выполнить DROP ROLE: 
DROP ROLE name;

Для удобства есть обертки createuser и dropuser, позволяющие выполнить создание и удаление ролей в терминале: 
createuser name
dropuser name 

Для того, чтобы посмотреть существующие роли, необходимо смотреть системный каталог pg_roles:
select rolname from pg_roles;
 
Метакоманда psql \du - также показывает существующие роли. 

Чтобы выполнить стартовую загрузку систем БД только что инициированная система всегда содержит одну предопределенную роль. Эта роль - всегда суперпользователь, и по умолчанию (можно изменить при запуске initdb) будет иметь то же самое имя, что и пользователь ОС, инициировавший кластер БД. Обычно, это роль postgres. Чтобы создать больше ролей необходимо подключиться под этой ролью.

Каждое подключение к серверу бд выполняется с использованием имени определенной роли, и эта же роль определяет начальные привилегии доступа для команд, выполняемых в рамках этого подключения. Имя роли, которое будет использоваться
для конкретного подключения к базе данных, указывается клиентом, который инициирует запрос на подключение в зависимости от приложения.

Например, psql использует настройку -U чтобы указать роль при подключении. Многие приложения предполагают по умолчанию роль с названием текущего пользователя ОС (createuser, psql). Часто очень удобно иметь соответствие по именам между ролями и пользователями ОС. 

Множество ролей БД для заданного клиентского подключения определяется настройкой клиентской аутентификации. Клиент не ограничен тем, чтобы подключаться под ролью не совпадающей с пользователем ОС, также, как и логин пользователя не должен совпадать с его действительным именем.
