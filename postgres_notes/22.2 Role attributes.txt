--
Роль БД может иметь определенные атрибуты, определяющие её привилегии и взаимодействие с системой аутентификации клиента. 

- привилегия логина 
	Только те роли, у которых есть атрибут LOGIN могут использовать свое название для подключения к БД. Роль с атрибутом LOGIN можно рассматривать как то же самое, что и пользователь БД. Чтобы создать роль с привилегией логин, нужно выполнить: 
		CREATE ROLE name LOGIN;
		CREATE USER name;
		
	CREATE USER то же самое что и CREATE ROLE за исключением того, что CREATE USER включает LOGIN по умолчанию, в отличии от CREATE ROLE. 
	
- статус суперпользователя 
	Суперпользователь БД проходит все проверки доступов, за исключением права залогиниться. Эту привилегию следует использовать с осторожностью: лучше выполнять большинство работ под ролью, которая не является суперпользователем. Для создания нового суперпользователя БД следует выполнить^ 
		CREATE ROLE name SUPERUSER;
	
	Необходимо выполнять под ролью, являющейся суперпользователем.

- создание бд 
	Роли необходимо явно выдать разрешение на создание БД (кроме суперпользователей, тк они обходят все проверки разрешений). Следует выполнить: 
		CREATE ROLE name CREATEDB;
		
- создание ролей 
	Роли необходимо явно выдать разрешение на создание других ролей (кроме суперпользователей). Следует выполнить: 
		CREATE ROLE name CREATEROLE;
	Роль с привилегией CREATEROLE может изменять и удалять другие роли тоже, как и определять членство в них. Но при этом для того чтобы создавать, изменять, удалять или изменять членство роли суперпользователя - требуется статус суперпользователя. 
	
- выполнение репликации 
	Роли необходимо явно выдать разрешение на инициация потоковой репликации (кроме суперпользователя). У этой роли также должно быть разрешение LOGIN. Для этого необходимо выполнить: 
		CREATE ROLE name REPLICATION LOGIN; 
		
- пароль 
	Пароль значимый, если способ аутентификации клиента требует у пользователя указания пароля при подключении к БД. Пароль требуется методам аутентификации password и md5. Пароли БД отличаются от паролей ОС. Они указываются с помощью:
		CREATE ROLE name PASSWORD 'string';
		
Атрибуты роли можно изменить после её создания при помощи ALTER ROLE. См документацию по командам CREATE ROLE, ALTER ROLE. 

Считается хорошей практикой создать роль, у которой есть атрибуты CREATEDB, CREATEROLE, но она не является суперпользователем - и использовать роль для конфигурирования БД и ролей. 

У роли также могут быть свои собственные значения по умолчанию для множества из настроек рантайма (см. документацию). Например, если хочется отключить сканирование индексов каждый раз при подключении, то следует использовать: 
	ALTER ROLE myname SET enable_indexscan TO off;

Такая инструкция сохранит настройку, но не выставит её сразу же. В последующих подключениях под этой роли будет выполняться SET enable_indexscan TO off перед тем как запустится сессия. Чтобы удалить специальную настройку следует выполнить: 
	ALTER ROLE rolename RESET varname;
	
Следует понимать, что такие настройки бесполезны для ролей без атрибута LOGIN - тк они никогда не вызовутся.
